# Session Bootstrap — MANDATORY before your first response

**Your first action in any session is to execute the `session-init` skill at `skills/session-init/SKILL.md` — before producing any user-facing message.** "Session start" means the first agent turn of any conversation, including conversations resumed from a context summary. A casual user message ("hi", a direct question) is a session-start signal, not a request for a casual reply.

This rule supersedes conversational instinct. Bootstrap first; address the user's actual content within or after the bootstrap output.
